Tools

Tools let agents invoke Julia functions during a conversation. The @tool macro inspects a function signature and generates JSON Schema metadata that LLMs use to decide when and how to call it. At runtime, invoke_tool executes the function and returns a result the LLM can consume.

Abstract Types

AgentFramework.AbstractTool — Type

Base type for all tool types.

FunctionTool

AgentFramework.FunctionTool — Type
FunctionTool <: AbstractTool

Wraps a Julia function as an AI-invocable tool with JSON Schema metadata.

Fields

  • name::String: Tool name (used in LLM function calling).
  • description::String: Human-readable description for the LLM.
  • func::Function: The underlying Julia function.
  • parameters::Dict{String, Any}: JSON Schema for the function parameters.
  • strict::Bool: Whether strict parameter validation is enforced.

Examples

# Using the @tool macro
@tool function get_weather(location::String, unit::String="celsius")
    "Get the current weather for a location."
    return "Sunny, 22°C in $location"
end

# Manual construction
t = FunctionTool(
    name = "get_weather",
    description = "Get weather for a location",
    func = my_func,
    parameters = Dict(
        "type" => "object",
        "properties" => Dict(
            "location" => Dict("type" => "string", "description" => "City name"),
        ),
        "required" => ["location"],
    ),
)

Tool Macro

AgentFramework.@tool — Macro
@tool function name(args...)
    "Description string"
    ...
end

Define a FunctionTool from a Julia function definition. The first string literal in the function body is used as the tool description. Parameter types are converted to JSON Schema automatically.

Example

@tool function search_web(query::String, max_results::Int=5)
    "Search the web and return results."
    # ... implementation
end
# Creates: search_web::FunctionTool

Schema and Invocation

AgentFramework.tool_to_schema — Function
tool_to_schema(tool::FunctionTool) -> Dict{String, Any}

Convert a FunctionTool to the OpenAI function-calling JSON schema format.

tool_to_schema(tool::HandoffTool) -> Dict{String, Any}

Generate the function calling schema for a handoff tool. Handoff tools take no parameters — the LLM simply calls them to transfer.

AgentFramework.invoke_tool — Function
invoke_tool(tool::FunctionTool, arguments::Dict{String, Any}) -> Any

Invoke a tool with parsed arguments. Arguments are matched to function parameters by name. Tracks invocation count and enforces maxinvocations / maxinvocation_exceptions limits.

invoke_tool(tool::FunctionTool, arguments_json::AbstractString) -> Any

Invoke a tool with JSON-encoded arguments.

Tool Collections

AgentFramework.normalize_tools — Function
normalize_tools(tools) -> Vector{FunctionTool}

Normalize a collection of tools to a flat vector of FunctionTool.

AgentFramework.find_tool — Function
find_tool(tools::Vector{FunctionTool}, name::String) -> Union{Nothing, FunctionTool}

Find a tool by name in a collection.

Utilities

AgentFramework.is_declaration_only — Function

Whether this tool is declaration-only (no implementation to execute).

AgentFramework.parse_result — Function
parse_result(value) -> String

Convert a raw tool return value to a string suitable for the LLM. Handles strings, dicts, vectors, Content objects, and arbitrary types.

AgentFramework.reset_invocation_count! — Function
reset_invocation_count!(tool::FunctionTool)

Reset invocation and exception counters to zero.
