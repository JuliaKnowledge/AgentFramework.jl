Evaluation

The evaluation framework provides tools for testing agent and workflow quality. Define EvalItem test cases with expected outputs, run them through an agent or workflow, and inspect EvalResults for pass/fail summaries and individual scores.

Core Types

AgentFramework.AbstractEvaluator — Type

Protocol for evaluation providers.

Any type implementing evaluate(evaluator, items, eval_name) and having a name field satisfies this protocol. See LocalEvaluator for a reference implementation.

AgentFramework.LocalEvaluator — Type

Evaluation provider that runs checks locally without API calls.

Implements the evaluator protocol. Each check function is applied to every item. An item passes only if all checks pass.

Example

local = LocalEvaluator(
    keyword_check("weather"),
    tool_called_check("get_weather"),
)
results = evaluate(local, items)
AgentFramework.EvalNotPassedError — Type

Raised when evaluation results contain failures.

Data Types

AgentFramework.EvalItem — Type

A single item to be evaluated.

conversation is the single source of truth. query and response are derived via the split strategy.

AgentFramework.EvalResults — Type

Results from an evaluation run by a single provider.

AgentFramework.EvalItemResult — Type

Per-item result from an evaluation run.

AgentFramework.EvalScoreResult — Type

Result from a single evaluator on a single item.

AgentFramework.ExpectedToolCall — Type

Expected tool call for correctness evaluation.

AgentFramework.CheckResult — Type

Result of a single check on a single evaluation item.

Conversation Splitting

AgentFramework.ConversationSplitter — Type

Strategy for splitting a conversation into (querymessages, responsemessages).

Any function f(conversation::Vector{Message}) -> (Vector{Message}, Vector{Message}) satisfies this interface. Built-in strategies are SPLIT_LAST_TURN and SPLIT_FULL.

AgentFramework.SPLIT_LAST_TURN — Function

Split at the last user message (default). Everything up to and including the last user message is the query; everything after is the response.

AgentFramework.SPLIT_FULL — Function

Split after the first user message. The first user message (and any preceding system messages) is the query; the entire remainder is the response.

AgentFramework.split_last_turn — Function

Split at the last user message (default). Everything up to and including the last user message is the query; everything after is the response.

AgentFramework.split_full — Function

Split after the first user message. The first user message (and any preceding system messages) is the query; the entire remainder is the response.

AgentFramework.split_messages — Function

Split the conversation into (querymessages, responsemessages). Resolution order: explicit split, then item.splitstrategy, then SPLITLAST_TURN.

AgentFramework.per_turn_items — Function

Split a multi-turn conversation into one EvalItem per user turn. Each item has cumulative context up to that turn.

Query and Response Extraction

AgentFramework.eval_query — Function

Get the user query text from an EvalItem (derived via split strategy).

AgentFramework.eval_response — Function

Get the agent response text from an EvalItem (derived via split strategy).

Result Inspection

AgentFramework.eval_passed — Function

Number of passing results.

AgentFramework.eval_failed — Function

Number of failing results.

AgentFramework.eval_total — Function

Total number of results (passed + failed).

AgentFramework.all_passed — Function

Whether all results passed with no failures or errors.

AgentFramework.raise_for_status — Function

Raise EvalNotPassedError if any results failed or errored.

AgentFramework.is_error — Function

Return true if the evaluation result represents an error during evaluation.

AgentFramework.is_passed — Function

Return true if the evaluation result represents a passing assertion.

AgentFramework.is_failed — Function

Return true if the evaluation result represents a failing assertion.

Built-in Checks

AgentFramework.keyword_check — Function

Check that the response contains all specified keywords.

Returns a check function for use with LocalEvaluator.

Example

check = keyword_check("weather", "temperature")
AgentFramework.tool_called_check — Function

Check that specific tools were called during the conversation.

mode can be :all (every tool must be called) or :any (at least one).

Example

check = tool_called_check("get_weather", "get_flight_price")
AgentFramework.tool_calls_present — Function

Check that all expected tool calls were made (unordered, extras OK).

Uses item.expected_tool_calls — checks names only, not arguments.

AgentFramework.tool_call_args_match — Function

Check that expected tool calls match on name and arguments.

Subset matching on arguments — extra actual arguments are OK.

Running Evaluations

AgentFramework.make_evaluator — Function
make_evaluator(fn; name=nothing) -> EvalCheck

Wrap a plain function as an EvalCheck for use with LocalEvaluator.

The function's parameter names determine what data it receives from the EvalItem. Supported parameter names: query, response, expected_output, expected_tool_calls, conversation, tools, context.

Return Bool, Number (≥0.5 = pass), Dict with score or passed key, or CheckResult.

Examples

# Simple boolean check
length_check = make_evaluator(; name="length_check") do response
    length(response) < 2000
end

# Score-based check
overlap = make_evaluator(; name="overlap") do query, response
    words = split(lowercase(query))
    matches = count(w -> occursin(w, lowercase(response)), words)
    return matches / max(length(words), 1)
end
AgentFramework.evaluate — Function

Evaluate a batch of items. Must be implemented by evaluator providers.

AgentFramework.evaluate_agent — Function
evaluate_agent(; agent, queries, evaluators, ...) -> Vector{EvalResults}

Run an agent against test queries and evaluate the results.

Arguments

  • agent: An agent instance (must support run_agent).
  • queries: Test query string(s).
  • expected_output: Ground-truth expected output(s), one per query.
  • expected_tool_calls: Expected tool calls per query.
  • responses: Pre-existing AgentResponse(s) to evaluate without running the agent.
  • evaluators: One or more evaluator instances or check functions.
  • eval_name: Display name for the evaluation run.
  • context: Optional grounding context.
  • conversation_split: Split strategy applied to all items.
  • num_repetitions: Number of times to run each query (default 1).

Returns

A Vector{EvalResults}, one per evaluator provider.

AgentFramework.evaluate_workflow — Function
evaluate_workflow(; workflow, evaluators, ...) -> Vector{EvalResults}

Evaluate a multi-agent workflow with per-agent breakdown.

Arguments

  • workflow: The workflow instance.
  • workflow_result: A completed WorkflowRunResult (post-hoc mode).
  • queries: Test queries to run through the workflow (run + evaluate mode).
  • evaluators: One or more evaluator instances or check functions.
  • eval_name: Display name for the evaluation.
  • include_overall: Whether to evaluate the workflow's final output.
  • include_per_agent: Whether to evaluate each sub-agent individually.
  • conversation_split: Split strategy applied to all items.
  • num_repetitions: Number of times to run each query (default 1).
AgentFramework.WorkflowAgent — Type
WorkflowAgent <: AbstractAgent

An agent that wraps a Workflow and exposes it via the standard agent interface. Runs the workflow with the input messages and returns the workflow's output as an AgentResponse.

Only output events from the workflow are converted to agent response messages. Other workflow events (logging, checkpointing) are ignored.

Fields

  • workflow: The workflow to wrap (any type supporting run_workflow).
  • name: Agent name.
  • description: Optional agent description.
  • id: Unique agent identifier.
  • context_providers: Optional context providers for the agent.
