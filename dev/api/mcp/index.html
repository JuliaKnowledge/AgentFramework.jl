MCP (Model Context Protocol)

MCP integration allows agents to discover and invoke tools hosted by external MCP servers over stdio or HTTP transports. Use connect! to establish a connection, list_tools to discover available tools, and mcp_tools_to_function_tools to convert them into FunctionTool instances the agent can use directly.

Client Types

AgentFramework.AbstractMCPClient — Type

Abstract base for MCP clients.

AgentFramework.StdioMCPClient — Type
StdioMCPClient <: AbstractMCPClient

MCP client using subprocess stdio transport.

Example

client = StdioMCPClient(command="npx", args=["-y", "@modelcontextprotocol/server-filesystem", "/tmp"])
connect!(client)
tools = list_tools(client)
result = call_tool(client, "read_file", Dict("path" => "/tmp/test.txt"))
close_mcp!(client)
AgentFramework.HTTPMCPClient — Type
HTTPMCPClient <: AbstractMCPClient

MCP client using HTTP transport (Streamable HTTP).

Example

client = HTTPMCPClient(url="http://localhost:8080/mcp")
connect!(client)
tools = list_tools(client)
close_mcp!(client)

Tool Conversion

AgentFramework.mcp_tool_to_function_tool — Function
mcp_tool_to_function_tool(client, tool; tool_name_prefix=nothing) -> FunctionTool

Convert an MCPToolInfo to an AgentFramework FunctionTool.

Supports structuredContent fallback: if tool result has no text content, falls back to structuredContent field. Tool names are normalized and optionally prefixed for deduplication.

AgentFramework.mcp_tools_to_function_tools — Function

Convert MCP tools to AgentFramework FunctionTool objects, with optional name prefixing.

AgentFramework.load_mcp_tools — Function

Connect to MCP server and return all tools as FunctionTools.
