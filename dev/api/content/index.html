Content

The content system provides a polymorphic representation of message payloads. Every Message carries a vector of Content items, each tagged with a ContentType discriminator. Factory functions such as text_content and function_call_content make it easy to construct content items without manipulating the struct directly.

Core Types

AgentFramework.AbstractContent — Type

Base type for all content variants.

AgentFramework.Content — Type
Content

Unified content container covering all content variants.

Use the constructor functions (text_content, data_content, function_call_content, etc.) rather than constructing directly.

Core fields

  • type::ContentType: The content variant.
  • text::Union{Nothing, String}: Text payload (for TEXT, TEXT_REASONING).
  • uri::Union{Nothing, String}: URI reference (for URI).
  • media_type::Union{Nothing, String}: MIME type (for DATA, URI).
  • call_id::Union{Nothing, String}: Tool call ID (for FUNCTIONCALL, FUNCTIONRESULT).
  • name::Union{Nothing, String}: Function/tool name (for FUNCTION_CALL).
  • arguments::Union{Nothing, String}: JSON-encoded arguments (for FUNCTION_CALL).
  • result::Any: Tool result value (for FUNCTION_RESULT).
  • annotations::Union{Nothing, Vector{Annotation}}: Content annotations.
  • additional_properties::Dict{String, Any}: Extension properties.
AgentFramework.ContentType — Type
ContentType

Enum of all content variant types, matching the Python ContentType literal.

AgentFramework.detect_media_type_from_base64 — Function
detect_media_type_from_base64(data::String) -> Union{String, Nothing}

Detect the media type of base64-encoded binary data by examining magic byte prefixes. Returns a MIME type string or nothing if unrecognised.

Examples

detect_media_type_from_base64("iVBORw0KGgoAAAANSU...")  # "image/png"
detect_media_type_from_base64("JVBERi0xLjQ...")          # "application/pdf"
detect_media_type_from_base64("dW5rbm93bg==")            # nothing
